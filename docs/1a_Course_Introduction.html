MADA Fall 2019 - Course Introduction

Andreas Handel

2019-08-05

Introduction

Welcome to the class Modern Applied Data Analysis (MADA)! The course is listed as EPID8060E/BIOS8060E. It doesn’t matter if you are enrolled under the EPID or BIOS label. I will refer to the course as MADA or EPID8060 interchangably. I think on eLC it is loaded as BIOS8060E.

Module Overview

The course is divided into modules. Each module consists of one or several units. This first module provides a brief introduction to the topic, and an overview of the course. I first describe my objectives, goals, and philosophy, followed by the logistic details of how everything will (hopefully) work.

This module also provides a few motivating examples of real world data analyses that I hope you will be able to do once you complete the course.

Module Learning Objectives

The specific learning objectives for this introductory module are:

  • Know what this course is all about.
  • Know how this course is set up and what you are expected to do.
  • Get motivated toward learning and doing data science.
  • See what’s possible with R and what you’ll be able to do yourself soon.

Course Learning Objectives

The main goal for this course is for you to learn the whole process of performing a data analysis project. This starts with identifying a suitable question-data pair, to getting, cleaning and exploring the data, all the way to fitting statistical models and producing materials that communicate what you did and found. A second goal is to introduce you to some modern analysis approaches that these days often go by the name of ‘Machine Learning’. Finally, a related goal is to introduce you to a set of tools that allow for a modern, reproducible workflow of your analyis.

The specific learning objectives that I hope you will achieve by going through this course are:

Design

  • Given a data set, define the questions that can be answered and formulate and implement suitable analytic approaches.
  • Given a data science question, determine the type of data and analytic approach needed to answer it.

Processing

  • Know how to obtain, process and analyze data from a variety of different sources.
  • Be knowledgeable about data types and standards and what analysis methods to apply to specific situations.
  • Be able to thoughtfully and critically assess strengths and weaknessess of specific data sets and process and analyze the data appropriately.

Analysis

  • Critically compare and evaluate the strengths and weaknesses of different data analysis approaches.
  • Judge the appropriateness of different approaches for specific questions and data sets and know how to apply an appropriate analytic approach.
  • Design and implement successful data analyses using state-of-the-art analysis software to translate data to information and knowledge that leads to actionable insights.

Communication

  • Summarize analysis results in ways that provide actionable conclusions and that are easily understandable by different audiences, such as lay persons, decision makers, and expert colleagues.
  • Assess the strengths and weaknesses of different formats for representing the results of data analyses.

Tools

  • Explain the importance of workflow, project management and reproducibility tools, and know how to use those tools.
  • Be comfortable using R and Github to do data analysis in a reproducible manner.
  • Be able to quickly learn how to use new software and tools, figure out how to get help when stuck, and make it work for you.

Overall

  • Judge the usefulness and appropriateness of data analyses described in the primary research literature.
  • Learn to look yourself over the shoulder and critically assess what you are doing, what assumptions you make by doing certain things, and if and how you can justify these.

Course Prerequisites

Nominally, for this course some R coding skills (e.g. as obtained in our EPID 7500 class), as well as basic knowledge of statistics at the level of fitting linear or logistic models to data (e.g. as obtained in our BIOS 7010 and 7020 courses) are expected. If you don’t have any of the pre-requisite R coding or statistics knowledge, you can still take the class, but you need to be prepared to spend some extra time and effort to fill any gaps. This will be especially true for the R coding part. Some students of you likely have pretty solid previous R experience, while others might have little to none. That means if you are in the latter category, expect to spend extra time getting up to speed with R. I believe it’s doable and worth the effort, but you need to be prepared for it. For instance we’ll spend one module on R Basics. For those with good R knowledge, this might be a 2 hour refresher, for those with none that week might be a 20+ hour committment. There will be plenty of help from myself, classmates and the internet if you end up getting stuck with some of the coding, but your effort and committment are still required.

Course Philosophy

I describe my thoughts on teaching and learning a bit more in the completely optional Teaching and Learning document. As a summary, here are my goals, promises and expectations for this course.

  • I will try to maximize the rewards you get by hopefully teaching useful material and trying (through online interactions) to help you learn as much as possible material that is directly useful and rewarding to you.

  • The environment for this class strives to be motivating and non-threatening. As such, I’ll make you work hard, and expect you to do the assigned tasks by the deadlines, but in the end, I usually don’t grade hard - unless you fail to keep up your end of the agreement and don’t put in the work.

  • I expect you to be self-motivated and committed to the material and class, and put in the effort needed to succeed.

  • This class is ‘open everything’. You can use the internet, ask your classmates, myself and others, get help from wherever you can. I trust you will find the right balance of getting help when you need it while still putting in enough effort to experience real learning.

  • I will not perform any ‘policing’ to try to prevent you from taking shortcuts (i.e. not doing work yourself). The class contains graded assessments with deadlines, but those are meant to help you stay on track. If you somehow cheat - and cheating will be easy - you are mainly cheating yourself out of learning. I hope you realize that this is pointless and defeats the purpose of why you take the class.

  • Overall, I will try to to provide you with a good balance of carrots and sticks to make it doable for you to commit your effort to this class.

I hope this course is going to be useful, interesting, fun and also interactive. Online courses are always a bit tricky to with interaction/participation. I hope we can create something online that feels like a classroom. Please participate, ask questions, etc. The more you engage in the course, the more you’ll get out of it.

Course Setup

  • The course is split into modules, each with one or several units. Each module will usually be covered in a week. The schedule document gives an outline. This schedule might change, so check frequently.
  • Each module consists of one or several units/documents containing a mix of things I wrote, writings or videos by others, exercises, etc.
  • All the material for this course is either on or accessed through this course website or (for quizzes) on eLC. Most material is not yet available, it will be added/unlocked as the course proceeds.
  • Your grade is made up of online quizzes (20%), discussion/participation (20%), homework/small projects (20%) and a course long project (40%).
  • Most modules have assessments (i.e. things you need to do for which you are graded). Due dates are usually Friday 5pm the week of the module, some might be at other times, e.g. the beginning or middle of the week. Each module contains a document with instructions and deadlines for the assessments.
  • The Projects section on the website with information for the small projects and the semester-long class project.
  • Syllabus and Schedule documents are in the General Information section of the course website. The former document contains a lot of basic logistic/legal stuff, the latter provides an outline of the course content (subject to change).

Course Tools

This is a brief overview of the tools we will be using for this class. The next module, which you can and should start right after finishing this one, describes all these tools in more detail and gives instructions on how to set them all up.

  • We will use the R software in this course.
  • We will also use RStudio (a graphical frontend to R).
  • We will use a lot of tools available within R/Rstudio, such as Rmarkdown and related features.
  • We will use Git/Github for several projects.
  • We will use eLC for discussions, help and quizzes.

Assessments

Every module will have an Assessment document. This document describes what work you need to do that will be graded. Most weeks, there is an online quiz and some discussion question. Sometimes, there is a small project, there is also a course-long project.

Online quizzes

Those quizzes are on eLC. They tend to be around 5-10 fairly easy questions. The main purpose of the quizzes is to ensure you worked through the assigned material. Quizzes are timed (usually 20min). You will have enough time to look up something you forgot but remember where it was, but you won’t have enough time to start the quiz and then try to find the answers in the materials. Therefore, before taking a quiz, make sure you do all the assigned reading, video watching, exercises, coding, etc. for the module. Sometimes I tell you to write down a specific answer. This means it’s likely going to be asked in the quiz.

Small projects

These projects are part of a single module, and are due at the last day of the module. They are described in more detail in the Projects section. Currently planned are 2 TidyTuesday projects.

Class project

This is a semester-long data analysis project broken into pieces with several deadlines. The Projects section and the Schedule document provide more details.

Discussion and Participation

At times, I ask you to post specific assignments to the discussion boards, and reply to students. I also encourage general participation on the discussion boards. This can take any form, such as asking questions, answering questions, posting links to interesting content, providing feedback, etc. The more, the better 😄.

Course Resources

We’ll be drawing on a lot of different resources. I usually point to specific resources when a certain topic is introduced. For convenience, I compiled a list with most of them in the Resources section of the class website.

Getting help

I do not expect you to figure it all out yourselves. You will get stuck, and are encouraged to seek help.
