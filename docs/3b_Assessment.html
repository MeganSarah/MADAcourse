MADA Fall 2019 - Course Introduction Assessment

Andreas Handel

2019-07-31

Overview

For this module, you need to do a small project that shows you have Rmarkdown and Github running and it will also introduce you to your classmates.

For those of you who are new to Github/R/Rmarkdown, this project makes you do a lot of things that you might not be familiar with. I know that this might be time consuming and also might feel a bit intimidating. It’s partly unavoidable and partly on purpose. You need to learn how to quickly get up to speed with all kinds of new tools that come your way. So practicing it is a good idea. You are welcome to draw on any sources for help that you want (online, classmates, instructor, etc). I’m confident with a bit of trial and error, you’ll get it to work.

Github setup

  • If not done so, create a Github account, email me your Github user name, wait for the invitation to join the class organization (epid8060fall2019) and join.
  • Once you are part of the organization, go to its website and create a new repository named YOURGITHUBNAME-introduction. You can see mine in there, which is called andreashandel-introduction. You can leave the description empty. Choose if you want the repository to be private or public. Check the box Initialize this repository with a README. Leave the other options at none.
  • Click on your new repository. Go to Settings in the top right. Under the Options tab in settings, scroll down to Github Pages and switch it to master from none. Don’t choose a theme.
  • Clone the repository to your local computer (using Gitkraken or whatever software/method you decided to use for this). I recommend that you make a folder somewhere on your computer called epid8060fall2019 and have all class repositories (of which this is the first one) as sub-folders. Remember that these folders should not be synced with some other software (e.g. Dropbpox).

RStudio project setup

  • Open RStudio on your computer. Choose File, New Project, Existing Directory and then choose as directory the location on your computer where you cloned your repository in the previous step, i.e. the folder/repository called YOURGITHUBNAME-introduction. If things work, you should see in the bottom right window of RStudio (at least) 2 files, one with .Rproj ending and a README.md.
  • Click to open the README.md file. It will only contain the name of your repository. Change the headline and call it “Introducing myself” (or something like that) and underneath write something like “This website contains a short introduction of Your Name.”

RMarkdown content creation

For this part of the task, the book R Markdown: The Definitive Guide and the RStudio RMarkdown website are likely useful.

  • Choose File, New File, R Markdown. Give it a title like YOUR NAME Introduction, keep output format as HTML. A new file with some template content should show up. You can leave the top of the content in there, delete what comes after the first heading (## R Markdown).
  • Update the YAML header (the information in the block starting/ending with —). Specifically, choose a layout theme. To figure out how to do that, check section 3.1.4 (as of this writing) of the R Markdown book referenced above. You are welcome to add further information to the YAML header (e.g. date, author name, etc).
  • Write a short post using R Markdown in which you introduce yourself. Talk a bit about yourself, your background, training, research interests. Let me/us know what kind of statistics, programming, data analysis experience you already have. I’d also be curious to know what you most hope to learn in this course. And then talk about (at least one) interesting personal fact(s) about you.
  • Include a picture of yourself. To do so, place the picture in the repository and link it from the Rmarkdown file.
  • Finally, include a link to a picture/video/website/etc that is somehow related to data analysis and that you find really cool/interesting/weird/etc. Provide a few explanatory sentences.
  • If you want, feel free to get creative and include other things. You can play with RMarkdown if you want, e.g. you can try to include some table or a video or…

Save the file you just created as index.Rmd into the repository folder. You need to give it this name, otherwise the Github website display won’t work. Then knit the file, you should see an index.html file show up.

Sending updates to Github

  • Make sure the 2 files (README.md and especially index.Rmd/index.html) look the way you want. Make changes until everything works. Once you are happy with things, close RStudio.
  • Go to the program you use for Github (I will assume that it’s GitKraken, if you use something else the details might vary). I the top right of Gitkraken, there should be a notification about changed files. Click view changes, then Stage. Write a commit message, commit the changes. You should see the master with the computer symbol be above the one with some random logo. That means your local repository is ahead of the one on github.com. To get them in sync, you click the push button. If things work, the two symbols should now be in the same line.
  • Go back to github.com and to your repository website. Under Settings and Github Pages it should give you a URL at which your new website lives. Something like https://epid8060fall2019.github.io/andreashandel-introduction/ (with my name switch with yours). Go to that website and it should show the introduction you just wrote.

Sharing your website

  • Go to eLC to the discussion forum called Introduction and write a short post with a link to your website/repository you created.
  • Also, if you don’t already, upload a picture of yourself to eLC. It’s nice to see faces during the online discussions there.
  • As you read your classmates introductions, feel free to comment/reply using the eLC discussion board.

You need to have your introductory website created and the link posted by the final date of the module (see syllabus). I encourage you to do it sooner, so we can get to know each other quickly!

Additional Information

  • If you want to keep making changes to your introduction, the workflow is first opening the repository in Gitkraken and making sure your local copy is up-to-date (this is important if you collaborate with someone or you are working on multiple local computers). If the remote is ahead of the local, do a pull. Once both are at the same stage, you can open your project in RStudio by double-clicking the name of the project (ends in .Rproj). Then make all edits you like. Make sure to knit your Rmd files. Once you are done with edits, close Rstudio, go to Gitkraken, stage/commit/push your changes back to the remote server.
  • If you like how fast you were able to create your own small website and want to learn more, 2 good resources are the blogdown book and the Rmarkdown website chapter in the Rmarkdown book. For my group website, I use blogdown/hugo and a theme called Academic. You can look at all the files here. The course website is simpler and uses the RStudio/Rmarkdown website setup. You can look at all these files here.
  • You can get started with either a blogdown website or a simple Rmarkdown website through File, New Project, New Directory and then choosing the option you want.
